# Hegel Salon Product Page

## 一句话

Hegel Salon 是一个把黑格尔式论证、原典阅读、现实判断、附件理解与浏览器代理放进同一工作流里的中文思想工作台。

## 它是什么

市面上大多数“哲学型 AI”要么只是模仿口吻，要么只是给出通用知识摘要。

Hegel Salon 想做的是另一种东西：

- 它把黑格尔当作工作方法，而不只是风格皮肤。
- 它让现实判断题带着历史形式与类比边界进入回答，而不是停在公共评论。
- 它允许用户把 PDF、Excel、图片和网页任务直接塞进同一条推理链。

换句话说，它不是“和黑格尔聊天”，而是“在一个黑格尔式工作台里思考问题”。

## 面向谁

### 1. 阅读者

适合想从原典、概念和论证结构出发读黑格尔的人，而不是只看二手概述的人。

### 2. 研究者

适合需要处理文本、引文、版本、附件材料和现实判断的用户。

### 3. 产品与策略工作者

适合想用更高强度概念框架来分析现实问题，而不满足于公共评论模板的人。

## 核心价值

### 原典约束

回答链会尽量让概念判断受黑格尔语料、版本、引文核验和来源层级约束。

### 概念压力

系统会倾向于把问题拆成：

- 对象是什么
- 常见理解错在哪里
- 真正矛盾在哪里
- 更强规定是什么

### 现实判断

现实政治与当代问题不会只给新闻评论式观点，而会尝试：

- 规定现实对象
- 调入历史形式
- 标出类比边界
- 再下判断

### 工作流一体化

你不需要在“问问题”和“处理材料”之间切换：

- 上传 PDF
- 上传 Excel
- 上传图片
- 让它打开网页
- 让它读取页面
- 再继续同一条对话

## 产品能力

### 1. Hegel 对话

面向概念问题与文本问题的主链路。

### 2. 附件理解

直接处理：

- PDF
- 表格
- 文本
- 图片

### 3. Browser Agent

内置浏览器代理，支持网页层的 computer use。

### 4. 历史引用模块

面向现实判断题的增强层。

### 5. 持续优化

不是只靠一次性 prompt 微调，而是保留失败样本、生成优化 playbook、持续回灌。

## 当前体验

你可以把它理解成一个黑金风格的“思想工作台”：

- 左侧是 Hegel Salon 的视觉入口
- 中间是主对话区
- 右侧可以打开材料与来源
- 顶部可以打开 computer use
- 下方可以直接带附件提问

## 和普通聊天产品的区别

| 维度 | 普通聊天产品 | Hegel Salon |
|---|---|---|
| 风格 | 通用助手风格 | 黑格尔式概念压力 |
| 引文 | 通常弱约束 | 强调核验与来源层级 |
| 现实题 | 倾向公共评论 | 引入历史形式与边界 |
| 附件 | 常常割裂 | 同一推理链处理 |
| 优化 | Prompt 为主 | 失败记忆 + 批量优化 |

## 当前边界

Hegel Salon 已经能工作，但并非所有部分都已经“完成”。

目前要诚实承认的边界有：

- 并不是所有回答都达到严格学术级质量
- 形式逻辑与史学严审仍在持续优化
- 90 分以上优化链已经跑通，但还没有稳定达标
- 现实判断题的历史对照还需要更多具体制度案例

所以它今天最准确的产品定位是：

一个已经可用、但仍在快速强化中的研究型智能工作台。

## 为什么值得发布

因为它已经跨过了“概念 demo”的阶段，进入了“可运行产品原型”的阶段。

它不是只有一个聊天框，而是同时拥有：

- 可运行的 Web 产品
- 原生 Android 壳
- 真实附件链
- 真实浏览器代理
- 真实日志与优化系统

这让它在 GitHub 上不只是一个想法，而是一个正在成长的系统。

## 未来版本

下一步最重要的升级方向是：

- 把现实判断的历史对照做得更具体
- 把 90 分优化做成更稳定的自动闭环
- 把优化进度做成可视化面板
- 把 Android 版做成正式发布包
